## 1. Fixed RLS Infinite Recursion on `profiles`

**Problem:** The "Profiles: same-university read" policy queried the `profiles` table to get the current user's `university_id`, causing infinite recursion.

**Fix:** Replaced the subquery with a JWT metadata read:
```sql
create policy "Profiles: same-university read"
  on profiles for select
  using (
    university_id = (auth.jwt() -> 'user_metadata' ->> 'university_id')::uuid
  );
```

**Why it works:** `university_id` is stored in `user_metadata` at signup, so middleware and policies can read it from the JWT without touching the `profiles` table.

## 5. Program Importer (`scripts/import-programs/`)

Built a Node/TypeScript importer to scrape the official [Study Program Handbooks](https://constructor.university/student-life/registrar-services/study-program-handbooks) page and store normalized programmes in Supabase.

**Schema:** `supabase/programs_table.sql`
- `programs` table: `name`, `slug` (unique), `level` (enum: bachelor/master), `school`, `source_url`, `is_active`
- Unique constraint on `(name, level)`
- RLS: authenticated read only

**Importer logic:**
- Fetches the Drupal-based page and parses `li.ck-list-marker-bold > strong` elements (the real CSS class used for program entries)
- State machine: tracks `h3` (bachelor/master boundary) → `h4` (year heading or school heading) → `li.ck-list-marker-bold` (program)
- Filters to 2023+ only
- Deduplicates by `(name, level)` — same program appearing in multiple year sections produces one row
- Skips minors (logged)
- Upserts via `onConflict: "name,level"`

**Result:** 25 unique programmes imported — 18 bachelor, 7 master.

**Run:** `npm run import:programs`

**Fixed during run:**
- First attempt parsed 0 programs — wrong selector (`ul > li` instead of `li.ck-list-marker-bold`)
- RLS blocked upsert with publishable key — fixed by adding `NEXT_PUBLIC_SUPABASE_SERVICE_ROLE_KEY` to `.env.local` and updating importer to fall back to it

## 7. CampusNet Course Catalogue Importer (`scripts/import-campusnet/`)

Built a BFS crawler + importer for the [CampusNet](https://campusnet.constructor.university/) public course catalogue (no login required).

**Schema:** `supabase/campusnet_tables.sql`

| Table | Purpose |
|---|---|
| `instructors` | Unique instructor names |
| `campusnet_courses` | Deduplicated academic modules (unique by `module_number` or `name`) |
| `course_offerings` | Semester-specific offering per module component |
| `course_offering_instructors` | Junction: offerings ↔ instructors |

Two partial unique indexes on `campusnet_courses`:
- `ON campusnet_courses (module_number) WHERE module_number IS NOT NULL`
- `ON campusnet_courses (name) WHERE module_number IS NULL`

**Crawler logic:**
1. Fetches nav page → extracts semester links from `.nav .depth_2.linkItem` where title matches `Spring|Fall YYYY` and year ≥ 2023
2. BFS per semester: follows `.auditRegNodeLink` links until reaching pages with `.eventTable`
3. Parses `.tbsubhead` rows → module header (module number + name extracted via regex `^([A-Z]{1,6}-\d{3}) (.+)`)
4. Parses `.tbdata` rows → course offering (offering number like CH-101-A, course type, instructors)
5. Upserts: instructors → courses → offerings → junction rows

**Resilience:**
- 600ms rate limit between requests
- 3 retries with exponential backoff
- 15s fetch timeout
- Graceful skip + log on any row failure
- `Prof. Professor Test` filtered out (Supabase placeholder)

**Run:** `npm run import:campusnet`
