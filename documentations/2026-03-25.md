## 2. Reddit-Style Upvote / Downvote on Reviews

**Schema:** `supabase/campusnet_review_votes.sql`
```sql
create table campusnet_review_votes (
  review_id  uuid references campusnet_course_reviews(id) on delete cascade,
  user_id    uuid references profiles(id) on delete cascade,
  vote       smallint not null check (vote in (1, -1)),
  primary key (review_id, user_id)
);
```
RLS: authenticated read; own insert/update/delete.

**Server action** `app/actions/review.ts` ‚Äî `voteReview(reviewId, vote: 1 | -1)`:
- Fetches existing vote for current user
- Same vote ‚Üí delete (toggle off)
- Different vote ‚Üí upsert

**UI (`ReviewCard` in `course-detail-tabs.tsx`):**
- Upvote / downvote buttons hidden on own review
- Optimistic state update via `useState` + `useTransition` ‚Äî UI updates instantly without waiting for server
- Highlighted state (blue) when user has voted

## 4. Professor Name Search on Courses Page

Extended `app/(main)/courses/page.tsx` search to match professor names.

**Logic (runs in parallel with main course query):**
1. `instructors.ilike('name', '%q%')` to find matching instructors
2. Walk join chain: `course_offering_instructors` ‚Üí `course_offerings` ‚Üí campusnet course IDs
3. Fetch those courses without school filter (so a professor's courses from all schools appear)
4. Merge and deduplicate with main results
5. JS-side keyword filter skipped when instructor match found
6. Count line shows "taught by or matching 'X'" when instructor was matched

## 6. Resources Tab ‚Äî Full Implementation

**Design:** Implemented from Stitch design. Upload button moved to top. "Top Contributor" section removed.

**`UploadResourceSheet` ‚Äî multi-file upload:**
- `files: File[]` state with multi-select file input
- File list with per-file remove buttons before submitting
- Sequential upload loop with progress indicator ("Uploading 2 of 3: notes.pdf")
- Auto-title from filename (extension stripped)
- Shared type/description across all files in one upload session
- On complete: `router.refresh()` (client-side, no `revalidatePath` needed)

**`ResourceCard` component:**
- Download button (opens `file_url` in new tab)
- Trash button on own resources with inline confirmation
- Client-side delete using browser Supabase client (`createClient` from `lib/supabase/client`)
- Calls `onDeleted()` ‚Üí `router.refresh()` on success

**Why client-side for upload + delete:** Server actions were not reliably forwarding auth cookies for storage/insert operations, causing RLS errors. Browser Supabase client uses the session cookie directly, making `auth.uid()` reliable in RLS policies.
